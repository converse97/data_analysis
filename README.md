# Xiaohongshu Public Search Crawler MVP

一个使用 Playwright + Supabase 的小红书公开搜索结果采集 MVP。

本项目只做低频公开页面采集：

- 不使用逆向接口
- 不抓包
- 不绕过平台限制
- 使用手动登录后保存的浏览器登录态
- 每条保存之间延迟 500ms
- 默认目标采集 100 条候选笔记，达到目标后停止

## Requirements

- Node.js 18+
- Supabase 项目
- 可访问小红书网页的本地浏览器环境

## Install

```bash
npm install
npx playwright install chromium
```

复制环境变量示例：

```bash
copy .env.example .env
```

填写 `.env`：

```bash
SUPABASE_URL=your-supabase-url
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

注意：`SUPABASE_SERVICE_ROLE_KEY` 权限很高，只应保存在本地 `.env`，不要提交到 Git。

## Supabase Table

在 Supabase SQL Editor 中执行：

```sql
create table public.xhs_notes (
  id bigint generated by default as identity primary key,
  keyword text not null,
  title text,
  url text unique,
  author_name text,
  like_count text,
  cover_url text,
  raw_data jsonb,
  crawled_at timestamptz default now()
);
```

## Login

首次运行需要手动登录并保存登录态：

```bash
npm run login
```

脚本会打开 `https://www.xiaohongshu.com`，请在浏览器中手动完成登录。登录完成后回到终端按回车，会生成：

```text
xhs-state.json
```

该文件包含本地浏览器登录态，已被 `.gitignore` 忽略。

## Crawl

默认采集最多 100 条公开搜索结果候选：

```bash
node crawl.js Patagonia
```

也可以明确指定目标数量：

```bash
node crawl.js Patagonia --limit=100
node crawl.js 露营 --limit=100
```

如果页面加载结果不足 100 条，可以适当提高最多滚动次数：

```bash
node crawl.js Patagonia --limit=100 --scrolls=30
```

参数说明：

- `--limit=100`：最多处理 100 条候选笔记。
- `--scrolls=30`：最多滚动 30 次。默认 20 次。
- 每次滚动后等待 2 秒。
- 每条写入 Supabase 后等待 500ms。

脚本会：

1. 读取命令行关键词。
2. 使用 `xhs-state.json` 登录态打开搜索页。
3. 访问 `https://www.xiaohongshu.com/search_result?keyword=关键词`。
4. 低频滚动页面，直到达到目标数量或滚动次数上限。
5. 采集公开可见的笔记候选信息。
6. 写入 Supabase 表 `xhs_notes`。
7. 以 `url` 作为唯一键，重复 URL 自动跳过。

写入字段：

- `keyword`
- `title`
- `url`
- `author_name`
- `like_count`
- `cover_url`
- `raw_data`
- `crawled_at`

## Analyze

从 Supabase 读取某个关键词的数据，生成拍摄参考报告：

```bash
node analyze.js Patagonia
```

也可以指定最多分析多少条：

```bash
node analyze.js Patagonia --limit=200
```

报告会生成到 `reports/` 目录，内容包括：

- 标题高频词
- 风格线索
- 场景线索
- 单品线索
- 拍摄方案草案
- 点赞较高的参考笔记
- 最近采集的笔记链接

## Compliance Notes

这个 MVP 的边界是“低频公开页面采集”。建议：

- 不采集私信、登录后非公开内容、手机号、地址、身份证、精准位置等敏感信息。
- 不绕过验证码、风控、登录限制或反爬限制。
- 不做并发采集，不高频刷新。
- 不把采集数据用于骚扰、画像、自动化营销或侵犯用户权益的用途。
- 如果平台提示访问异常、要求验证或限制访问，应停止采集。

小红书页面结构可能变化，`crawl.js` 使用公开 DOM 中的链接、图片和文本做保守提取。如果页面结构调整，可能需要更新选择器。
